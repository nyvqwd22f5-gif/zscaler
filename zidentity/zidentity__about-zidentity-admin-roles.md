# About ZIdentity Admin Roles
Source: https://help.zscaler.com/zidentity/about-zidentity-admin-roles
PDF: https://help.zscaler.com/pdf/com/en/1499256.pdf

Zscalerâ€™s role-based administration enables you to assign ZIdentity users or user groups with admin roles and permissions and control their level of access to the ZIdentity Admin Portal.
There are two predefined admin roles:
- Super Admin: System role with full permissions. You can assign this role to other admins if required.
- View Only Admin: System role view only permissions.
You cannot delete the predefined roles.
Ensure that you have [added users](/zidentity/adding-users) and [user groups](/zidentity/adding-user-groups) to the ZIdentity Admin Portal before assigning necessary admin roles.
For ZIdentity tenants who have migrated to Experience Center, you must have at least the View Only permission for the Users and User Groups modules to submit a Zscaler Support ticket from within Experience Center.
Role management provides the following benefits and enables you to:
- Assign users and user groups as admins to manage tasks in the ZIdentity Admin Portal.
- View all the configured admins, their access levels, and functional scope.
About the Roles Page
On the Roles page (Administration > Entitlements > ZIdentity Roles), you can do the following:
- [Add an admin role](/zidentity/adding-zslogin-admin-roles).
- View a list of roles configured for your organization. For each role, you can view:
- **Name**: The name of the role.
- **Description**: The description of the role, if available.
-
View the predefined role's permissions.

You cannot delete predefined roles.
- [Edit or delete role](/zidentity/editing-or-deleting-items).
- Search for a role.
