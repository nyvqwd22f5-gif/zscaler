# About IP Locations
Source: https://help.zscaler.com/zidentity/about-ip-locations
PDF: https://help.zscaler.com/pdf/com/en/1499126.pdf

This article describes how to manage IP locations on the IP Locations page. These IP locations can be utilized for defining various sign-on policies.
IP Locations provide the following benefits and enable you to:
- Categorize multiple traffic egress points within your organization that access the ZIdentity service.
- Manage [sign-on policies](/zidentity/about-sign-on-policies) for users based on their locations.
About the IP Location Page
On the IP Locations page (Administration > Environment > IP Locations), you can do the following:
- View a list of configured IP locations. For each IP location, you can see:
- **Name**: The IP location name.
- **Country**: The IP location country.
- **IP Address**: The IP address of the location.
- **Source**: The source from where the IP location is added.
- [Add an IP Location](/zidentity/adding-ip-locations).
- [Import new IP locations, or modify existing IP locations, using a CSV file](/zidentity/importing-ip-locations-csv-file).
- Bulk delete IP locations. This **Actions** option is visible only when you select more than one location.
- Reset your search results.
- Search for an IP location.
- Select IP locations to perform bulk operations on them. You must select more than one IP location to view the **Actions** option.
- [Edit a configured IP location](/zidentity/editing-or-deleting-items).
- [Delete a configured IP location](/zidentity/editing-or-deleting-items).
