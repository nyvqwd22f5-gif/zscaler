# About ITDR API Token Management
Source: https://help.zscaler.com/itdr/about-itdr-api-token-management
PDF: https://help.zscaler.com/pdf/com/en/1531688.pdf

Zscaler ITDR supports the OpenAPI specification for Endpoint Agents, Threat Detection, Orchestrate, Investigate, and Settings features.
API tokens provide the following benefits and enable you to:
- Programmatically manage event logs, configure threat detection polices, orchestrate rules, etc.
- Make API calls.
About the ITDR API Tokens Page
On the API Tokens page (Orchestrate > API Tokens), you can do the following:
- [Add an API token](/itdr/creating-api-token).
- View a list of API tokens. For each API token configured, you can view:
- **Name**: The name of the API token.
- **Description**: The description of the API token.
- **Roles**: The role assigned to the API token.
- **Created By**: The ID of the user who created the API token.
- **Created On**: The date and time when the API token was created.
- **Enabled**: Indicates if the API token is enabled or disabled.
- **Status**: Indicates if the API token is active or expired.
- Access the API documentation.
- [Edit or delete an API token](/itdr/editing-or-deleting-api-token).
