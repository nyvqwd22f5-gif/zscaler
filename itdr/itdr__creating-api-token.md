# Creating an API Token
Source: https://help.zscaler.com/itdr/creating-api-token
PDF: https://help.zscaler.com/pdf/com/en/1531692.pdf

You can add and configure API tokens for authentication and making API calls in Zscaler ITDR.
To create an API token:
- Go to **Orchestrate** > **API Tokens**.
-
Click **Add API Token**.
[See image.](#itdr-add-api-token-option)
-
In the **API Token Details** window:
- **Enabled**: Select to enable the API token.
- **Name**: Enter the name of the API token.
- **Description**: Enter the description of the API token.
- **Roles**: Select a [role](/itdr/about-users-roles#itdr-user-role-about-role) for the API token from the drop-down menu.
- **Expiration (in days)**: Enter the API token expiration period in days.
You cannot modify the expiration period after the API token is created. You must create a new token after it expires.
[See image.](#itdr-api-token-details)
- Click **Submit**.
-
In the **API Token generated** window, copy the API token.
[See image.](#itdr-copy-api-token)
You cannot retrieve the API token after you close the window.
-
Click **Close**.
The API token is created.
[See image.](#itdr-api-token-added)
