# Cloud Asset Types and Asset Categories
Source: https://help.zscaler.com/zpc/cloud-asset-types-and-asset-categories
PDF: https://help.zscaler.com/pdf/com/en/1443016.pdf

An asset refers to a cloud resource (virtual machine, database, storage bucket, etc.). The assets are grouped by Asset Types and Asset Categories.
- Asset Types are services defined by the cloud service provider (CSP). For example, EC2 instances in Amazon Web Services (AWS) or virtual machines (VMs) in Microsoft Azure.
- Asset Categories are ZPC-specific aggregations for similar asset types across all CSPs. For example, the VMs and EC2 instances belong to the Compute Asset category.
Each asset category includes multiple asset types.
Asset Types
ZPC collects asset configuration metadata across all CSPs and provides an inventory of all assets deployed on your cloud infrastructure (i.e., assets not protected by ZPC and assets that are protected by ZPC) using security policies. ZPC provides an inventory of asset types on Microsoft Azure, AWS, Google Cloud Platform (GCP), and Kubernetes Security Posture Management (KSPM).
The following CSP-defined asset types are supported:
- [AWS](#aws-assettype)
- [Microsoft Azure](#azure-assettype)
- [GCP](#gcp-assettype)
- [Kubernetes](#kspm-assettypes)
ZPC supports the following CSP-defined asset types for public exposure protection:
- [AWS](#aws-assettype-public-exposure)
- [Microsoft Azure](#azure-assettype-public-exposure)
- [GCP](#gcp-assettype-public-exposure)
- [Kubernetes](#kspm-assettypes-public-exposure)
