# Usage Guidelines for Zero Trust Device Segmentation Gateways
Source: https://help.zscaler.com/zero-trust-branch/usage-guidelines-zero-trust-device-segmentation-gateways
PDF: https://help.zscaler.com/pdf/com/en/1529002.pdf

Zero Trust Device Segmentation gateways are installed in an organization’s network and are dedicated to the organization’s traffic. The Zscaler Operations team performs the following tasks on the organization's Zero Trust Device Segmentation gateways:
- Provides the required hardware and software for the initial installation of the Zero Trust Device Segmentation gateways.
- Ensures that the Zero Trust Device Segmentation gateways are suitably packed for shipment.
- Upgrades or replaces the Zero Trust Device Segmentation gateways to maintain the operational standards of Zscaler. Zscaler will ship the hardware to the organization. It is the organization's responsibility to set up the replacement hardware.
- Maintains the technical refresh status of the Zero Trust Device Segmentation gateways.
- Provides remote upgrades of software on the Zero Trust Device Segmentation gateways on a schedule that conforms with Zscaler's standard release and maintenance practices.
Zero Trust Device Segmentation gateways remain the property of Zscaler during the applicable subscription term.
Organization's Responsibilities
For Zscaler Operations to service, manage, and maintain the Zero Trust Device Segmentation gateways, Zscaler requires the organization to:
- Provide a local point of contact for delivery acceptance and assist Zscaler with any specific requirements for delivery.
- Ensure that the Zero Trust Device Segmentation gateways are not moved from one physical location to another without prior approval from Zscaler.
- Provide a continuous, uninterrupted, and suitable power supply.
- Implement and maintain temperature, humidity, and other environmental conditions that Zscaler recommends.
- Implement suitable surge protection devices.
- Ensure that each Zero Trust Device Segmentation gateway has internet connectivity.
- Ensure that the Zero Trust Device Segmentation gateways are used to provide Zscaler service only and are not used for any other purpose.
- Ensure that only Zscaler staff access or service the Zero Trust Device Segmentation gateways. Neither the organization nor any third party can make any repair attempts or other changes to the Zero Trust Device Segmentation gateways unless they have explicit approval from Zscaler.
- Manage all network-related issues.
- Use the Zero Trust Device Segmentation gateways only in accordance with instructions prescribed by Zscaler and only with hardware and software provided by Zscaler. Also, ensure that the Zero Trust Device Segmentation gateways are installed and operated in accordance with the specifications and recommendations provided by Zscaler.
- Comply with the firewall access requirements, if you use a firewall to control access to the products on the Zero Trust Device Segmentation gateways.
- Connect the Zero Trust Device Segmentation gateways to the appropriate network resources recommended by Zscaler and provide the network resources and the necessary IP addresses to run the service.
If there are any issues, you can receive Zero Trust Device Segmentation services through [Zscaler Support](/contact-support). Zscaler Support works with you to troubleshoot and diagnose the issues remotely. If the issue cannot be resolved remotely or if it is a hardware fault, Zscaler Support might decide to replace the Zero Trust Device Segmentation equipment.
Zscaler offers comprehensive return and advanced replacement services for Zero Trust Device Segmentation gateways. For details and for information about initiating hardware replacement, see [Return Merchandise Authorization Process and Policy](/zero-trust-branch/return-merchandise-authorization-process-and-policy).
To learn more about the Zero Trust Device Segmentation gateway, see the detailed [device requirements](https://www.zscaler.com/legal/zero-trust-device-segmentation-gateways).
