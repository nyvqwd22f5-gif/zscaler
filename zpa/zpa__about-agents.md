# About Agents
Source: https://help.zscaler.com/zpa/about-agents
PDF: https://help.zscaler.com/pdf/com/en/1531950.pdf

The Agents page gives insight into your organization's agent data. It allows you to monitor and analyze the data for all configured agents in your organization.
The Agents page provides the following benefits and enables you to:
- View a summary of agent data in your organization.
- View, monitor, and analyze agent data for a single workload.
- Take appropriate actions based on the agent data displayed.
- Discover insights about collective agent data.
About the Agents Page
On the Agents page (Microsegmentation > Agent Management > Agents), you can do the following:
- Refresh the Agents page to reflect the most current information.
- View a list of applied filters available from the current and previous user sessions. Applied filters must be saved to the user session first before they can be viewed. Use the drop-down menu to select the applied filters to view. To learn more, see [Using Tables](/zpa/using-tables).
- Show or hide the filter options.
- Filter the information by **Name**, **Admins Status**, **Connection Status**, **Agent ID**, **Current Software Version**, and **Private IPs**. By default, no filters are applied.
- [Add](/zpa/configuring-agents) an agent.
- View a list of all agents that are configured for your organization. For each agent, you can see:
- **Name**: The name of the agent.
- **Current Software Version**: The software version of the agent.
- **Connection Status**: The connection status of the agent, CONNECTED or DISCONNECTED.
- **Admin Status**: The admin status of the agent, ENABLED or DISABLED.
- Click an individual agent to view its complete information.
[See image.](#resource-info-window)
- [Edit an agent.](/zpa/editing-agents)
- Manually upgrade an agent.
- [Delete an agent.](/zpa/deleting-agents)
- [Modify the columns displayed in the table.](/zpa/using-tables)
- Display more rows or a different page of the table.
- Go to the** **[Agent Groups](/zpa/about-agent-groups-page) page to create and manage agent groups.
- Go to the** **[Agent Provisioning Keys](/zpa/about-agent-provisioning-keys-page) page to create and manage agent provisioning keys.
