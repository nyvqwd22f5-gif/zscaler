# Supported Versions & OS Compatibility for Microsegmentation
Source: https://help.zscaler.com/zpa/supported-versions-os-compatibility-microsegmentation
PDF: https://help.zscaler.com/pdf/com/en/1531938.pdf

To begin configuring [Microsegmentation policies](https://help.zscaler.com/zpa/about-microsegmentation-policies), you must first configure and deploy agents to your machine based on your OS.
To configure agents for your organization, see [About Agents](https://help.zscaler.com/zpa/about-agents) and[ Configuring Agents](https://help.zscaler.com/zpa/configuring-agents).
Minimum OS Version Compatibility
Each OS requires a specific compatible version to configure Microsegmentation.
| OS | Versions |
| --- | -------- |
| Windows | Server 2012 R2Server 2016Server 2019Server 2022 |
| Ubuntu Distribution | 16.04.7 LTS18.04.6 LTS20.04.6 LTS22.04.5 LTS24.04.2 LTS |
| Red Hat Enterprise Linux (RHEL) Distribution | RHEL 7.4 (3.10.0-693.el7.x86_64)RHEL 8RHEL 9 |
| Amazon Linux | AL2AL 2023 |
If you have the appropriate agent version and compatibility for your OS, then you have access to the following Microsegmentation capabilities in the ZPA Admin Portal:
- Dashboard
- Analytics
- Resource Management
- Policy
- Agent Management
