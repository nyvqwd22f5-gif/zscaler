# About Deception API Token Management
Source: https://help.zscaler.com/deception/about-deception-api-token-management
PDF: https://help.zscaler.com/pdf/com/en/1531427.pdf

Zscaler Deception supports the OpenAPI specification for Orchestrate, Miragemaker, Deceive, Landmine, Investigate, and Settings features.
API Tokens provide the following benefits and enable you to:
- Programmatically configure and deploy decoys, manage event logs and evidence, configure Miragemaker datasets and personalities, etc.
- Make API calls.
About the API Tokens Page
On the API Tokens page (Orchestrate > API Tokens), you can do the following:
- [Add an API token](/deception/creating-api-token).
- View a list of API tokens. For each API token configured, you can view:
- **Name**: The name of the API token.
- **Description**: The description of the API token.
- **Roles**: The [role](/deception/about-users-roles#deception-user-role-about-role) assigned to the API token.
- **Created By**: The ID of the user who created the API token.
- **Created On**: The date and time when the API token was created.
- **Enabled**: Indicates if the API token is enabled or disabled.
- **Status**: Indicates if the API token is active or expired.
- Access the API documentation.
- [Edit or delete an API token](/deception/editing-or-deleting-api-token).
