# Monitoring Data Explorer Views
Source: https://help.zscaler.com/zdx/monitoring-data-explorer-views
PDF: https://help.zscaler.com/pdf/com/en/1487211.pdf

Data Explorer offers the flexibility to build and organize your own customized views of applications and metrics. This flexibility can help provide context when analyzing and correlating data across your organization.
Make sure you meet the feature prerequisites to monitor Data Explorer views. To learn more, see [Configuring Data Explorer Views](/zdx/configuring-data-explorer-views).
To monitor Data Explorer views, go to **Analytics** > **Data Explorer**. The Data Explorer table provides the following information:
- **Name**: The name of the configured view.
- **Data Source**: Indicates the view is for End User Monitoring or Hosted Monitoring.
- **Applications**: The applications included within the view.
- **Probes**: The associated probes (if the Data Source is for Hosted Monitoring).
- **Metrics**: The metrics included within the view.
- **Aggregation**: The Aggregation type across all users, designated as Average, Minimum, or Maximum.
- **Created By**: The admin who created the view.
- **Last Updated On**: The date and time when the view was last edited or created.
![Table of Data Explorer views](/downloads/zdx/analytics/monitoring-data-explorer-views/zdx-de-table-headers2.png)
In addition to Search, the following actions are available within the Data Explorer table:
- [Create a Data Explorer view](/zdx/configuring-data-explorer-views).
- Click a name to display its Data Explorer view. For details about the interaction and format of Data Explorer charts, see [Viewing System-Generated Reports](/zdx/viewing-system-generated-reports).
- [Edit or delete a Data Explorer view](/zdx/configuring-data-explorer-views).
