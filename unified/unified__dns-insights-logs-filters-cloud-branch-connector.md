# DNS Insights Logs: Filters for Cloud & Branch Connector
Source: https://help.zscaler.com/unified/dns-insights-logs-filters-cloud-branch-connector
PDF: https://help.zscaler.com/pdf/com/en/1516891.pdf

Filters define the DNS traffic information that you view in your DNS Insights Logs. To learn more about logs, see [About Insights Logs](/unified/about-insights-logs-0).
You can select the following filters for your DNS Insights Logs:
- **AWS Account ID**: Use this filter to limit the data to traffic associated with the AWS account ID.
- **AWS Availability Zone**: Use this filter to limit the data to traffic associated with the availability zone where Cloud Connector for AWS is deployed.
- **AWS Region**: Use this filter to limit the data to traffic associated with the location of your AWS cloud account.
- **Azure Availability Zone**: Use this filter to limit the data to traffic associated with the availability zone where Cloud Connector for Azure is deployed.
- **Azure Region**: Use this filter to limit the data to traffic associated with the location of your Azure cloud account.
- **Azure Subscription ID**: Use this filter to limit the data to traffic associated with the Azure subscription ID.
- **Connector Group**: Use this filter to limit the data to traffic associated with the Cloud or Branch Connector group.
- **Connector Instance**: Use this filter to limit the data to traffic associated with the specific Cloud or Branch Connector instance.
- **Connector VM**: Use this filter to limit the data to traffic associated with the Cloud or Branch Connector virtual machine (VM) from which you want to view the DNS insights.
- **Connector Name**: The name of the Cloud or Branch Connector.
- **DNS Gateway Flag**: Use this filter to limit the data to traffic associated with the DNS gateway flag.
- **DNS Gateway Name**: Use this filter to limit the data to traffic associated with the DNS gateway name.
- **DNS Request Type**: Use this filter to limit the data to the traffic associated with a specific type of DNS request. Choose the request type from the list.
- **DNS Rule Name**: The rule that was triggered by the DNS request, response, or both.
- **Forwarding Type**: Use this filter to limit the data to traffic associated with the forwarding type. Choose the forwarding type from the list.
- **In Bytes**: Use this filter to limit the data to packets sent from the server to the client that were within a specific size range. Choose a predefined range or specify a custom range.
- **Location**: Use this filter to limit the data to a location's traffic. On the Locations page, select a location from the list of internet gateway locations. To find a specific location, use the Search function. You can choose to include or exclude certain locations.
- **Out Bytes**: Use this filter to limit the data to packets that the server received that were within a specific size range. Choose a predefined range or specify a custom range.
- **Platform**: Use this filter to limit the data to traffic associated with the hypervisor or the cloud platform from which you want to see the DNS insights.
- **Protocol Type**: Use this filter to limit data to either TCP or UDP traffic.
- **Request Action**: Use this filter to limit the data to a specific action that your DNS Control policy took.
- **Request Duration**: Use this filter to limit the data to the traffic associated with the specified request duration.
- **Requested Domain**: Use this filter to limit the data to the traffic associated with the domain for which DNS resolution was requested. Enter all or part of the domain in the text field and choose **Contains**, **Starts With**, **Ends With**, **Exact Match**, **Does Not Contain**, **Does Not End With**, **Not Null**, or **Is Null**.
